containerization with docker

a major goal of this project was for the "end" product to be as maintainable and replicable as possible. this was achieved by running each component of the system in individual, lightweight, and highly specialized docker containers. this architecture promotes separation of concerns, keeps each environment independent and uncluttered, and allows for quick & easy startup/teardown

extract, transform, & load (etl)

all data is sourced from nba.com via the nba-api python package and manipulated in python before being inserted into a database on my mariadb (fork of mysql) server
python etl code

database design/administration

the database was designed & normalized to be as close to third normal form as possible

http server/api

every interaction with jdeko.me is handled via a handwritten http server & api in go (formerly golang). this go application runs in its own indpenedent environment via docker.
docker networking is utilized and configured via docker compose to allow for communicaton between the api & database
